## 3. หน้า regional hub ภาคอีสาน

**สร้างแล้ว** — ไฟล์ `src/content/locations/ภาคอีสาน.md`

- **URL:** `/รับซื้อโน๊ตบุ๊ค/ภาคอีสาน/`
- **Title / H1:** ตามที่กำหนด (seoTitle + h1 ใน frontmatter)
- มีลิงก์ไปทุกจังหวัด 20 แห่ง และ FAQ สำหรับ hub

## 4. หน้า /พื้นที่ให้บริการ/

**อัปเดตแล้ว** (`src/pages/พื้นที่ให้บริการ/index.astro`):

- เพิ่ม **H2 รับซื้อโน๊ตบุ๊คภาคอีสาน** พร้อมเนื้อหาอธิบายการส่งรูปประเมินและการสอบถามพื้นที่ให้บริการ
- เพิ่ม **grid ลิงก์ 20 จังหวัด**
- แก้ส่วนลิงก์ภาคอีสานท้ายหน้าให้เป็น grid ครบ 20 จังหวัด + ลิงก์ไป hub

## 5. Internal links จังหวัดใกล้เคียง

ทำครบตาม mapping ที่กำหนดใน `scripts/isan-provinces-data.mjs` — แต่ละหน้ามีส่วน **จังหวัดใกล้เคียงที่ให้บริการ** เป็นลิงก์ markdown 3–6 จังหวัด (ไม่ลิงก์ครบทุกจังหวัดในทุกหน้า)

ทุกหน้าจังหวัดมีลิงก์ไป:

- `/` (รับซื้อโน๊ตบุ๊ค)
- `/รับซื้อโน๊ตบุ๊ค/`, `/รับซื้อโน๊ตบุ๊คมือสอง/`, `/เช็คราคาโน๊ตบุ๊ค/`, `/ตีราคาโน๊ตบุ๊ค/`, `/พื้นที่ให้บริการ/`
- `/รับซื้อโน๊ตบุ๊ค/ภาคอีสาน/`
- หน้าเงื่อนไข/แบรนด์ตามสเปก (เครื่องเสีย, เปิดไม่ติด, จอแตก, แบตเสื่อม, macbook, gaming)

กล่อง **Internal links** ใน `src/pages/รับซื้อโน๊ตบุ๊ค/[slug].astro` ปรับให้มีลิงก์ตามรายการที่ใกล้เคียงกับที่ร้องขอ (รวมหน้าแรกและพื้นที่ให้บริการ)

## 6. ผล npm run build

```
npm run build
```

**ผ่าน (exit code 0)** — สร้าง static routes รวมหน้าใหม่ (รวม `/รับซื้อโน๊ตบุ๊ค/ภาคอีสาน/` และจังหวัดอีสานทั้งหมด) และ sitemap ถูก generate โดย `@astrojs/sitemap`

## 7. Schema, CMS, เทคนิคอื่น

- **content.config.ts:** เพิ่ม field ออปชัน `seoTitle`, `h1`, `region` สำหรับ collection `locations`
- **`[slug].astro`:** location ใช้ `seoTitle` เป็น `<title>` (fallback ตามรูปแบบ @webuy), H1 จาก `h1` หรือรูปแบบมาตรฐาน **ส่งรูปเช็คราคาฟรี**
- **Decap CMS** (`public/admin/config.yml`): เพิ่มฟิลด์ SEO Title, H1, Region ใน collection locations
- **สคริปต์สร้างเนื้อหา:** `scripts/isan-build.mjs` + `scripts/isan-provinces-data.mjs` — รัน `node scripts/isan-build.mjs` เพื่อ regenerate ไฟล์ markdown ชุดเดียวกัน (ระวัง overwrite การแก้มือหลัง generate)
- **รูป featuredImage จังหวัดอีสาน:** แต่ละจังหวัดใช้ไฟล์ SVG แยกไฟล์ใน `public/images/locations/` (ชื่อ `isan-*.svg`) สร้างจาก `node scripts/gen-isan-location-svgs.mjs` — แมป slug → ชื่อไฟล์อยู่ที่ `ISAN_SVG_BASE` ใน `isan-provinces-data.mjs` หน้า hub ภาคอีสานใช้ `/images/locations/isan-region-hub.svg`

## 8. ความยาวเนื้อหาและการขยายต่อ

เป้าหมายเดิม ~2,500–3,500 คำต่อหน้า — ในรอบนี้ใช้ **draft แน่น** โครงสร้าง H2 ครบ + intro เฉพาะจังหวัด + อำเภอตามรายการ + FAQ 6–8 ข้อ + ตัวอย่างแนวทางประเมิน (ไม่ใช้คำว่าเคสจริง) โดยประมาณ **~1,000–1,800 คำไทยต่อหน้า** (แล้วแต่จังหวัด)

**แนะนำให้ขยายต่อเป็นพิเศษ (ใส่บริบทท้องถิ่น, รูปถ่ายจริง, คำถามเฉพาะธุรกิจในเมือง):**

- แทนที่ SVG ด้วยรูปถ่าย/ภาพโปรโมชัน **.webp ต่อจังหวัด** เมื่อมีสินทรัพย์จริง แล้วอัปเดต `featuredImage` ใน frontmatter (หรือปรับ `ISAN_SVG_BASE` + path ในลูปกำหนด `p.image` ให้ชี้ `.webp`)
- ทุกหน้า: เพิ่มย่อหน้าเชิงประสบการณ์ในพื้นที่ (โดยไม่อ้างรีวิวจริงหากไม่มีหลักฐาน), เพิ่ม FAQ เฉพาะอุตสาหกรรม/มหาวิทยาลัยในเมืองนั้น ๆ แบบไม่ซ้ำ template

## 9. คำแนะนำรูปจริงและรีวิว

- ปัจจุบันใช้ **SVG 1200×630** เป็นภาพหัวข้อ/OG ต่อจังหวัด (ไม่ใช่ภาพถ่ายสถานที่จริง) — เหมาะเป็น placeholder จนกว่าจะมีรูป `.webp`
- หากต้องการรูปเชิงพาณิชย์: ใส่เป็น `/images/locations/{ชื่อ}.webp` แล้วแก้ `isan-provinces-data.mjs` ให้ `p.image` ชี้ไฟล์นั้น (หรือแก้ frontmatter หลังรัน build โดยไม่รัน `isan-build` ทับ)
- เพิ่มรูปในเนื้อหา body (ก่อน/หลังส่วนอำเภอ) เฉพาะเมื่อมีลิขสิทธิ์และสถานที่จริง
- หากมีรีวิวจริงจากลูกค้า: ใส่ใน `src/content/reviews/` และอ้างอิงจากหน้าจังหวัดอย่างเป็นธรรมชาติ (ไม่แต่งรีวิว)
